# 腾讯云安全组放行清单

部署话图 T2G 时，需要在**腾讯云控制台 → 轻量应用服务器 → 防火墙**配置入站规则。

## MVP 试用期（HTTP + 8080）

| 应用类型 | 协议 | 端口 | 策略 | 备注 |
|---|---|---|---|---|
| 自定义 | TCP | 22 | 允许 | SSH 远程登录；建议限制来源 IP 为运维办公网 |
| 自定义 | TCP | 8080 | 允许 | 老师浏览器访问入口 |
| 自定义 | TCP | 443 | 允许 | 预留，HTTPS 上线后用 |

> 国内运营商对未备案的 80 端口会有干扰，所以**不开 80**；老师访问要带端口号 `:8080`。

## 上线 HTTPS 后（域名 + Caddy）

| 应用类型 | 协议 | 端口 | 策略 | 备注 |
|---|---|---|---|---|
| 自定义 | TCP | 22 | 允许 | SSH |
| 自定义 | TCP | 80 | 允许 | Caddy 自动重定向 HTTP→HTTPS（域名备案后） |
| 自定义 | TCP | 443 | 允许 | 主入口 |

切到 https profile 后，可以关闭 8080。

## 配置位置（截图位置说明）

1. 控制台首页 → 「轻量应用服务器」
2. 点击你的实例 → 左侧「防火墙」标签
3. 「添加规则」→ 选「自定义」→ 填上表中的协议/端口
4. 「来源」一般填 `0.0.0.0/0`（全网），SSH 建议改成你的固定 IP 段

## 验证

外网验证端口是否真的开了：

```bash
# 在你的本机（非服务器）执行
nc -vz <服务器公网IP> 8080
# 期望：Connection succeeded

curl -I http://<服务器公网IP>:8080
# 期望返回 200/304
```

如果 `nc` 超时但 `docker compose ps` 显示 frontend healthy，**100% 是安全组没配对**。

## 常见坑

- **轻量应用服务器有"防火墙"，CVM 有"安全组"** —— 两者名字不同但作用一样，按你实际购买的产品类型找
- 改完规则**立即生效**，不需要重启实例
- 单 IP 限制 SSH 时，如果你换了网络（出差、家里换 IP）会被锁在外面 → 留一个"我的 IP 实时检测"页面书签，或保留一条 `0.0.0.0/0` 的临时规则用于紧急访问
