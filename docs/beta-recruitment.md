# 内测招募与反馈跟踪 · v0.12.0

> 本文档是 V2 主线完成后开启老师试用的执行手册。目的：**用 2 周时间收 5-10 位老师的真实反馈**，为下一阶段决策（V2-C / V2-D / W12+）提供依据。

## 时间线（建议 2 周）

| 阶段 | 时间 | 关键动作 |
|---|---|---|
| **T0 · 冻结** | 第 0 天 | v0.12.0 部署到腾讯云 + 前端 Ctrl+F5 确认新版加载 |
| **T1 · 招募** | 第 1-3 天 | 联系 5-10 位老师，逐个发访问链接 + 教师手册 |
| **T2 · 试用** | 第 4-10 天 | 老师日常使用；产品方每日看 👍/👎 与 admin/feedback 接口 |
| **T3 · 收敛** | 第 11-14 天 | 汇总反馈 → 决策文档 → 下一里程碑 Plan |

## 招募对象画像（优先级）

1. **初中数学老师**（K12 目标用户）—— 3-5 位
2. **高中数学老师**（试探能否复用于圆锥曲线等 V3 场景）—— 1-2 位
3. **教研员 / 教辅编辑**（画图密集，反馈质量高）—— 1-2 位
4. **不熟悉 AI 的老教师**（考察易用性）—— 1 位

## 每日检查项（产品方，每日 10 分钟）

```bash
# 1. 服务活着
curl -s http://49.233.15.73:8080/api/health

# 2. 昨天用量
curl -s 'http://49.233.15.73:8080/api/admin/stats?days=1' | jq

# 3. 反馈（重点看 👎）
curl -s 'http://49.233.15.73:8080/api/admin/feedback?days=1' | \
  jq '.[] | select(.rating=="bad")'

# 4. 拿完整 JSONL（便于分析）
curl -s 'http://49.233.15.73:8080/api/admin/feedback.jsonl?days=7' \
  > /tmp/feedback-$(date +%Y%m%d).jsonl

# 5. 后端错误日志
docker logs t2g-backend --tail 200 2>&1 | grep -iE "error|traceback|refuse|solve_fail"
```

## 分类反馈的思考框架

收到 👎 后按 **原因 × 严重度** 归类：

| 原因分类 | 举例 | 处理策略 |
|---|---|---|
| **DSL 表达能力不够** | "我要画阴影区域，画不了" | 记入下一里程碑候选（需要新对象/约束类型） |
| **LLM 理解错** | 说"边长 3" 画成 "AB=3" 但应该是 "BC=3" | 加 few-shot 示例、改 prompt 引导 |
| **求解器给错解** | 镜像解、位置颠倒 | 加半平面约束或方位提示（W10 已有基础） |
| **渲染问题** | 标签重叠、圆太小、颜色不对 | 局部修 svg.py |
| **交互问题** | 找不到导出按钮、撤销不明显 | 前端优化 |
| **性能** | 太慢、卡 | 上 SSE 流式 / 加缓存 |
| **导出兼容** | 复制到 PPT 字丢了 | V2-C outline |

## 分析后的产出（第 14 天）

在 `docs/feedback-v0.12.md` 里生成：

1. **数据摘要**：总消息数、👍/👎 比、平均残差、平均延迟
2. **Top 5 问题**：按分类频次排序
3. **具体案例**：每个类型给 2-3 个真实老师的话与截图
4. **下一里程碑选题**：基于反馈量化的 3 个候选（预估工作量 + 收益）

## 与老师同步的节奏

- **T3 之前**：不主动打扰。除非老师主动问，或收到系统级 bug 报告
- **T7 中期**：可以发一条统一消息：「大家试用得怎么样？有什么想到的都欢迎发过来」
- **T14 收尾**：给每位老师一份「反馈总结 + 感谢」，让老师知道自己的话被听到了

## 应急处理

- **腾讯云宕机**：`./deploy/bootstrap.sh` 重跑；不行降级到 `docker compose down && up -d`
- **LLM Key 用光**：切到 `DEFAULT_PROVIDER=deepseek` 或 `zhipu`（`.env` 改一行 + `docker compose restart backend`）
- **数据库损坏**：从 COS 恢复最近备份（详见 `docs/operations.md` 第 4.2 节）
- **老师报安全问题**：如"我看到别人的会话"，立即 `docker compose down` 排查
